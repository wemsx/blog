自建api，走起！

September 2, 2022 · 1 min · wemsx

先放个效果：(假设没问题的话，每次刷新会变)

具体原理很简单，下面放的就是基本代码

<?php
//初始化随机数生成器种子，这行代码也可以删除
$seed = time();
//获取随机数
$num = rand(1,80);
//拼接图片地址
$picpath = "https://gcore.jsdelivr.net/gh/YOUR_NAME/YOUR_REPO/IMAGE_FOLDER/".$num.".jpg";
//重定位到图片
die(header("Location: $picpath"))

代码来源: 搭建自己的随机图API——DearXuan

原文建议是全部存Github，然后可以用JSD加速。这是最便宜最好的方案。但今天我尝试的时候发现貌似JSD突然不可用了，反正一直无法获取图片。最后干脆改成raw,githubusercontents.com 。注意这个地方不是github那个打不开的那个raw，后面加了一个s，这是一个开放的加速CDN。按道理来说，任何方式的云储存都可以作为图片储存，前提就是可以直链，可以自定义。而且不建议用付费的云储存，避免出现意外。

1-80是该文件夹下图片的序号。如果假定刚好有1～80共80张图片，那么就请将所有图片名称改成纯数字.jpg。（不是只能JPG，但不能同时存在多种格式。）

然后就可以啦！我是在vercel上部署的。当然也可以宝塔。vercel的话还要添加一个额外的 vercel.json。代码如下：

{
  "functions": {
    "api/vlts/jpg/*.php": {
      "runtime": "vercel-php@0.4.0"
    },
    "api/vlts/webp/*.php": {
      "runtime": "vercel-php@0.4.0"
    },
    "api/art/*.php": {
      "runtime": "vercel-php@0.4.0"
    },
    "api/zelda/*.php": {
      "runtime": "vercel-php@0.4.0"
    }
  },
  "routes": [
    { "src": "/zelda",  "dest": "/api/zelda/index.php" },
    { "src": "/art",  "dest": "/api/art/index.php" },
    { "src": "/vlts/jpg",  "dest": "/api/vlts/jpg/index.php" },
    { "src": "/vlts/webp",  "dest": "/api/vlts/webp/index.php" }
  ]
}

这里我比原文小小修改了一下，就是我的API不止一个。

对于Vercel部署，上面的没什么好说的，就是相当于用指定的runtime（这里就是vercel-php@0.4.0）运行指定目录的文件，只需要对应写就可以了。注意JSON格式！！！下面routes很有意思，最开始看到的项目的src是 "/(*)"，就是说你部署到Vercel后访问xxx.vercel.app时不管你后面加什么，它都会给你显示后面的dest对应路径的内容。这里我写的四个就对应四个不同的index.php文件，就相当于是四个不同的API。虽然简单不了多少，但至少满足了强迫症，爽！

至于访问速度还是有待提高的，但毕竟免费提供服务已经很不错了。而且不需要像上一个那样要把所有图片链接单独列成一个TXT。
